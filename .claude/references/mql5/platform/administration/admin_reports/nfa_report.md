# NFA Reports

U.S. regulator [National Futures Association](https://www.nfa.futures.org/) (NFA) has specific requirements concerning financial control and reporting for U.S.-registered brokerages. To simplify the compliance with the regulator's requirements, the NFA Reports is included in the platform standard delivery. It automatically generates a number of reports to be submitted to NFA.

The reports are generated according to the regulator requirements described in Forex Data Layout Overview and Implementation Reference Version 1.8.9. The applied format consists of text ASCII files using field separators similar to CSV. The format is used for easy parsing and analysis of reports on the NFA side.

The report itself is implemented as the NFA.Reports64.dll file combining the report and the server plugin. The plugin collects the necessary data and saves it in the trade server directory, while the report provides collected data in the appropriate format. The plugin should be [enabled](/en/docs/mt5/platform/administration/admin_reports/nfa_report#install_plugin) in the settings of the appropriate trading server. If the plugin is not enabled, report data will not be updated and the report will provide data only for the plugin's operation period.

The NFA.Reports64.dll file is included into the standard delivery and located in Reports directory of MetaTrader 5 trading server.

## Available reports

In accordance with the [NFA Rule 2-48](https://www.nfa.futures.org/news/PDF/CFTC/CR2_48FDMTradeReporting0810.pdf), the plugin generates the following report types:

-   Orderbook Changes — a list of order transactions (Buy/Sell market orders and pending orders).
-   Trade — list of transactions.
-   Customer Info — list of active clients.
-   Adjustments — list of transaction modifications (modification of an open price/volume of an open position or modification of an open price/close price/volume/profit of a closed position).
-   Ad Hoc Tick Data — list of ticks for a day.

The following types of reports are also generated by the plugin, but they do not contain any data since there is no corresponding functionality in the MetaTrader 5 platform. These reports are generated because they should still be submitted to the regulator even if there is no data.

-   Money Manager Info — a list of NFA-accredited account managers.
-   Customer/Manager Concordance — a list of customers to account manager concordance.
-   Market Events — a list of significant market events (e.g. a failure of a trade server).

All generated reports are placed in the directory of the server the plugin is installed for: \[trade server installation directory\]\\NFA\\Export\\\[trading day date\]. A trading day means the period from 17:00 (05:00 PM) of the the previous day to 17:00 (05:00 PM) of the current day in the EST time zone. For example:

-   An event that occurred on 13.12.2010 at 16:50 applies to the trading day of 13.12.2010.
-   An event that occurred on 13.12.2010 at 17:50 applies to the trading day of 14.12.2010.

In addition, all the time values given in the statement are converted to the EST time zone.

The detailed description of all reports is provided in the "[Reported data](/en/docs/mt5/platform/administration/admin_reports/nfa_report#data)" section below.

### Exporting stop loss and take profit data

Among other things, the Orderbook Changes and Trade reports should contain data on stop loss and take profit orders. However, in MetaTrader 5, they are represented as position fields/properties rather than separate orders. Therefore, when generating the reports, the plugin exports these fields as Stop and Limit orders on its own. An operation ID (ORDER\_ID) is generated for them in a special way:

-   Position take profit is exported as a Limit order with the ID \[Position ticket\]\_TP\_\[counter\]
-   Position stop loss is exported as a Stop order with the ID \[Position ticket\]\_SL\_\[counter\]

Here, the ticket is a position ID on the platform side, while the counter is a serial number of a stop loss or a take profit set for a specific position. Example:

1.  A stop loss is set to 1000 for a position with the ticket 12345. In the report, the operation is displayed as ORDER ADD 12345\_SL\_1.
2.  The stop loss is then removed. This operation is displayed as ORDER CANCELLED 12345\_SL\_1.
3.  A new stop loss for the position is set to 500. This operation is displayed as ORDER ADD 12345\_SL\_2.
4.  The stop loss is changed to 700. This change is displayed as ORDER MODIFY 12345\_SL\_2 in the report.

Thus, the counter value is changed only if a stop loss/take profit is removed and a new one is placed, which means an old order is removed, while a new one is placed. When changing the levels, it is assumed that the modification is applied to the same order.

## Plugin installation [#](nfa_report#install_plugin)

The appropriate plugin should be installed to collect the data used in the report. To install the plugin, enter [Plugins](/en/docs/mt5/platform/administration/admin_plugins) section in MetaTrader 5 Administrator. After the server with the installed plugin has been selected, click ![Add](/en/docs/mt5/platform/img/add_button.png "Add") Add.

![Configuring the plugin](/en/docs/mt5/platform/img/nfa_plugin_config.png "Configuring the plugin")

The following settings are available:

-   Symbol — sort by symbols. Symbols are separated with a comma. To specify symbols, you may also use the mask "\*" and the negation sign "!". For example, the value "EURUSD,GBPUSD" means that the plugin will collect the transaction details for EURUSD and GBPUSD symbols. The value "\*" is for all symbols. \*,!CFD\\\* value is for all symbols except those in the CFD group.
-   Groups — sort by groups. Groups are separated with a comma. To specify groups, you may also use the mask "\*" and the negation sign "!". For example, the value "demoforex,democfd" means that the plugin will collect the transaction details for users in "demoforex" and "democfd" groups. The value "\*" is for all groups. The default value is "\*,!demo\*,!contest\*" which means all groups except those whose names start with "demo" and "contest".
-   Forex Dealer Member ID — a seven-digit identifier of a broker given by NFA (FDM's NFA ID).
-   Server ID — trade server name specified in the report.
-   Base Directory — name of the directory where the database of transactions is saved for the report. The data is saved by days. This directory is automatically created in Reports\\NFA.Reports. In most cases, this parameter should save a default value.
-   SFTP Address — address for connecting to Secured FTP server the reports are to be [uploaded](/en/docs/mt5/platform/administration/admin_reports/nfa_report#send) to.
-   SFTP Port — port for connecting to Secured FTP server.
-   SFTP Login — login for connecting to Secured FTP server.
-   SFTP Password — password for connecting to Secured FTP server.
-   SFTP Private Key Path — path for the private key for authorization on Secured FTP server.
-   Report Time — report generation time (EST).
-   Send Time — time of sending reports to the SFTP server (EST).

## Report installation [#](nfa_report#install_report)

Reports are generated and sent to NFA by the server plugin. The report configuration stage is needed only to be able to request and view them via the Manager terminal.

To install the report, enter [Reports](/en/docs/mt5/platform/administration/admin_reports) section in MetaTrader 5 Administrator. After the server to install the report has been selected, click ![Add](/en/docs/mt5/platform/img/add_button.png "Add") Add.

![Configuring the report](/en/docs/mt5/platform/img/nfa_report_config.png "Configuring the report")

Each of the NFA Reports is configured separately. If you want to view the Addhoc Tick Data Requests report via the Manager terminal, create a separate configuration for it. The same should be done if you need the Orders report. There is no point in creating configurations for the Money Managers, Customer/Manager Concordance and Market Events reports since they always remain empty and are necessary only when reporting to NFA, which is performed by the plugin.

After adding a configuration, select the necessary report in the Type field and set the only additional parameter Base Directory for it. This is a path to the directory where the search for report data is performed. It should correspond to the path specified in the Base Directory parameter in the [plugin settings](/en/docs/mt5/platform/administration/admin_reports/nfa_report#install_plugin). If not a single plugin saves data at the specified path, the report remains empty. Multiple reports may take data from the same directory.

## Viewing the reports

To view the report request, use MetaTrader 5 Manager. Simply select the necessary report in the Server Reports section to retrieve data from the server. The report data can be sorted by group, logins, period and symbol mask.

Find out more about working with the reports in the appropriate section of MetaTrader 5 Manager documentation.

## Sending the reports to NFA [#](nfa_report#send)

To submit the reports to NFA, the appropriate files should be uploaded to the regulator's special SFTP server, except for Addhoc Tick Data Requests which is requested by NFA only if necessary. NFA Reports plugin is able to send reports to the server automatically. To achieve this, specify the sending time and the SFTP server connection data in its [settings](/en/docs/mt5/platform/administration/admin_reports/nfa_report#install_plugin).

To get SFTP access (address, login and password), contact NFA in accordance with Security Request Procedure described in Forex Data Layout Overview and Implementation Reference.

Apart from a login and a password for accessing SFTP, you also need to create a bunch of RSA keys (private/public) and send the public key to NFA. To do this, follow the stages described in Forex SFTP Connectivity Implementation Reference:

1.  On the [http://www.putty.org/](https://www.putty.org/) website, download the [puttygen.exe](https://the.earth.li/~sgtatham/putty/latest/x86/puttygen.exe) (PuTTY SSH key generation utility) application.
2.  Launch [puttygen.exe](https://the.earth.li/~sgtatham/putty/latest/x86/puttygen.exe) and select SSH-2 RSA key type.
3.  Click Generate and follow the application instructions.
4.  Click "Save public key" and specify the name of the public key file.
5.  Click "Save private key" and specify the name of the private key file.
6.  Send the public key to NFA for its subsequent installation on the SFTP server.

<table class="attentable" cellspacing="0" cellpadding="8" border="1"><tbody><tr class="attentable"><td class="attentable"><ul><li class="p_tableatten"><span class="f_tableatten">NFA requires a public key file name to contain Forex Dealer Member ID.</span></li><li class="p_tableatten"><span class="f_tableatten">The private key is an additional means of authentication on the SFTP server. Keep it in a safe place and do not submit it to anyone.</span></li><li class="p_tableatten"><span class="f_tableatten">When saving the private key, do not specify "passphrase". Otherwise, the plugin will not be able to automatically send reports to the SFTP server due to incompatibility of pagent.exe and Windows services.</span></li></ul></td></tr></tbody></table>

After NFA installs the public key on its SFTP server, you will be able to send reports via the plugin automatically. For configuration:

1.  Specify data for connecting to Secured FTP server in the [plugin settings](/en/docs/mt5/platform/administration/admin_reports/nfa_report#install_plugin) (address, login, password and path to the private key).
2.  Set report sending time in the plugin settings.

## Report data [#](nfa_report#data)

The contents of all reports are described in details in Forex Data Layout Overview and Implementation Reference. NFA Reports is fully compliant with the requirements. The general description is provided below for more convenience.

<table class="attentable" cellspacing="0" cellpadding="8" border="1"><tbody><tr class="attentable"><td class="attentable"><p class="p_tableatten"><span class="f_tableatten">The time in all reports is indicated according to the EST time zone.</span></p></td></tr></tbody></table>

### Orderbook Changes

-   ID — unique operation ID within a single report (for one day).
-   BATCH\_DATE — report date.
-   TRANSACTION\_DATETIME — operation date and time.
-   ORDER\_ID — order ticket/ID on MetaTrader 5 side.
-   FDM\_ID — broker ID in NFA.
-   MKT\_SEG\_ID — market segment ID. The field remains blank since there is no such entity in MetaTrader 5.
-   PRODUCT\_CODE — financial instrument name.
-   PRODUCT\_CAT — product category. C — call option, P — put option, F — Forex symbol, R — forward contract.
-   CONTRACT\_YEAR — forward/option contract execution year.
-   CONTRACT\_MONTH — forward/option contract execution month.
-   CONTRACT\_DAY — forward/option contract execution day.
-   STRIKE — strike option price.
-   OPTION\_TYPE — option type: standard or exotic.
-   TRANSACTION\_TYPE — transaction type: entry, modification, execution, etc.
-   ORDER\_TYPE — order type.
-   VERB — operation direction: buy or sell.
-   BID\_PRICE — Bid price at the time of operation.
-   ASK\_PRICE — Ask price at the time of operation.
-   QUANTITY — operation volume in the number of contracts. The value cannot be fractional. If the calculated volume has a fractional part, it will be rounded to the nearest whole number.
-   REMAINING\_QTY — volume of an unexecuted part of an order (filled only for partially executed orders). The value cannot be fractional. If the calculated volume has a fractional part, it will be rounded to the nearest whole number.
-   PRICE — Limit price for limit orders and option price.
-   STOP\_PRICE — stop price in orders.
-   STOP\_PRODUCT\_CODE — dependent product code. The field remains blank since there is no such entity in MetaTrader 5.
-   TRAIL\_AMT — trailing size. The field is always set to zero since a trailing stop works only on the client terminal side.
-   LIMIT\_OFFSET — distance to a limit price in trailing stop limit orders. The field is always set to zero since there is no such entity in MetaTrader 5.
-   DURATION — order type by expiration type: GTC, up to a given date, time, etc.
-   EXPIRY\_DATE — order expiration date.
-   ORDER\_ORIGIN — order source. SYSTEM — created by the platform, DLR — created by a dealer, CUST — created by a client.
-   MANAGER\_ID — capital manager ID. The field remains blank since there is no such entity in MetaTrader 5.
-   CUSTACCT\_ID — client account number.
-   SERVER\_ID — unique server ID (set in the [plugin settings](/en/docs/mt5/platform/administration/admin_reports/nfa_report#install_plugin)).
-   CUST\_GROUP — client category. The field remains blank since there is no such entity in MetaTrader 5.
-   LINKED\_ORDER\_ID — linked order ID (for example, for OCO orders). The field remains blank since there is no such entity in MetaTrader 5.
-   LINK\_REASON — reason an additional order is linked to a main one. The field remains blank since there is no such entity in MetaTrader 5.
-   OPEN\_CLOSE — currently not used, always remains blank.

### Trade

-   ID — unique operation ID within a single report (for one day).
-   BATCH\_DATE — report date.
-   TRADE\_ID — trade ticket/ID on MetaTrader 5 side.
-   ORDER\_ID — order ticket/ID on MetaTrader 5 side.
-   FDM\_ID — broker ID in NFA.
-   MKT\_SEG\_ID — market segment ID. The field remains blank since there is no such entity in MetaTrader 5.
-   TIME\_TRADEMATCH — trade execution time.
-   PRODUCT\_CAT — product category. C — call option, P — put option, F — Forex symbol, R — forward contract.
-   PRODUCT\_CODE — financial instrument name.
-   CONTRACT\_YEAR — forward/option contract execution year.
-   CONTRACT\_MONTH — forward/option contract execution month.
-   CONTRACT\_DAY — forward/option contract execution day.
-   STRIKE — strike option price.
-   VERB — operation direction: buy or sell.
-   BID\_PRICE — Bid price at the time of operation.
-   ASK\_PRICE — Ask price at the time of operation.
-   QUANTITY — operation volume in the number of contracts. The value cannot be fractional. If the calculated volume has a fractional part, it will be rounded to the nearest whole number.
-   REMAINING\_QTY — volume of an unexecuted part of a trade (filled only for partially executed trades). The value cannot be fractional. If the calculated volume has a fractional part, it will be rounded to the nearest whole number.
-   FILL\_PRICE — trade price.
-   CONTRA\_FILL\_PRICE — initial trade price at a liquidity provider side (if trades are sent to external systems).
-   SERVER\_ID — unique server ID (set in the [plugin settings](/en/docs/mt5/platform/administration/admin_reports/nfa_report#install_plugin)).
-   IMPLIED\_VOLATILITY — implied option volatility.
-   IB\_REBATE — broker markup for a trade.
-   COMMISSION — commission for a trade.

### Customer Info

-   BATCH\_DATE — report date.
-   CUSTACCT\_ID — client account number.
-   FDM\_ID — broker ID in NFA.
-   CUST\_NAME — customer's name.
-   CUST\_BIRTH\_DATE — customer's date of birth. The value is copied from the [client record](/en/docs/mt5/platform/administration/clients#personal) to which the trading account is linked.
-   CUST\_TYPE — customer type. ECP — counterparty, RETAIL — retail client, OMNIBUS — white label.
-   CUST\_GROUP — client category. The field remains blank since there is no such entity in MetaTrader 5.
-   COUNTRY\_TYPE — customer type by country. FOR — foreign,  DOM — local.
-   NET\_LIQUIDATING\_VALUE — account liquidation value.
-   ACCT\_OPEN\_DATE — account opening date.

### Adjustments

-   BATCH\_DATE — report date.
-   FDM\_ID — broker ID in NFA.
-   TRADE\_ID — trade ticket/ID on MetaTrader 5 side.
-   ADJUSTMENT\_DATETIME — adjustment time.
-   ADJUSTMENT\_TYPE — adjustment type. PRICE — price change, QTY — volume change, CASH — monetary outcome change.
-   PRICE — new price after an adjustment.
-   QUANTITY — new operation volume in the number of contracts.
-   CASH\_AMOUNT — amount charged/debited from an account.
-   NOTE — adjustment reason.

### Ad Hoc Tick Data

-   FDM\_ID — broker ID in NFA.
-   BATCH\_DATE — report date.
-   PRODUCT\_CODE — financial instrument name.
-   QUOTE\_DATETIME — quote date and time.
-   BID\_PRICE — Bid price.
-   ASK\_PRICE — Ask price.

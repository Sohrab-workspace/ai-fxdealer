# Nomupay

Nomupay is a global payments provider dedicated to helping brokers expand seamlessly across borders. The company combines regional expertise with a unified payments platform designed to simplify how businesses collect, convert and move money internationally. With the extensive network of local acquirers across Europe, the Middle East and Southeast Asia, brokers gain reliable access to hard-to-reach markets through a single integration, reducing operational complexity while increasing acceptance rates and customer reach.

The provider supports a wide range of currencies and payment methods, including PayPal, Alipay, and local payment systems.

To connect Nomupay payments, click 'Connect provider' in the [gallery](/en/docs/mt5/platform/administration/payments/payments_wallets) and fill out a short form. You can also use the registration form on the [provider's website](https://nomupay.com/application/). The company representative will contact you and will provide further instructions.

## Connection Credentials [#](payments_nomupay#api)

After registration, you will be given access to your personal account, where you can find details for connecting the provider. Open the 'Credentials' section. This section contains the so called channels — separate configurations within the payment system created to accept payments for a specific website, store, etc. Channels are created by Nomupay. Select the channel created for use with MetaTrader 5, then click the eye icon. Copy the Entity ID and Access Token parameters.

![Find connection details](/en/docs/mt5/platform/img/payments_nomupay_api.png "Find connection details")

Connection parameters are displayed only once. Be sure to store them in a secure location. If you need to retrieve the credentials again, please contact Nomupay support.

## Provider Configuration [#](payments_nomupay#configure)

Create a [payment gateway configuration](/en/docs/mt5/platform/administration/payments/payments_wallets), select Nomupay for the gateway and enter connection details: Entity ID in the Login field and Access Token in the Password field.

![Create a wallet](/en/docs/mt5/platform/img/payments_nomupay_common.png "Create a wallet")

In the "Type" field, only one general payment method is currently available: "Any". This means that the selection of the payment system or bank is made directly on the payment provider's page, to which the user is redirected from the trading terminal. The list of payment methods available to the user is defined at the channel level by Nomupay staff.

The provider supports deposit (top-up) operations only.

## System Features and Limitations [#](payments_nomupay#limitations)

Nomupay provider supports deposit (top-up) operations only. Withdrawals are not available.

## Setting Up the Environment [#](payments_nomupay#sandbox)

Nomupay supports operations in a test sandbox environment. In this environment, the provider simulates transaction processing so that you can check how payments work, without performing real money transactions. To connect to the sandbox environment, the company will provide you with a separate personal account, where you can obtain all the necessary integration details as described above. Additionally, enable the 'Sandbox' option in the wallet settings so that the platform marks all transactions within it as test operations.

![payments_nomupay_parameters.png](/en/docs/mt5/platform/img/payments_nomupay_parameters.png "payments_nomupay_parameters.png")

## Currency settings [#](payments_nomupay#currencies)

You can view the list of currencies available for your wallet in the "channel" properties in the Credentials section of your personal account.

![payments_nomupay_supported_currencies](/en/docs/mt5/platform/img/payments_nomupay_supported_currencies.png)

Add the required currencies on the MetaTrader 5 side to prevent clients from selecting unsupported currencies when making payments through the terminal. Also, set transaction amount limits in accordance with Nomupay's requirements.

![Specify the payment currency](/en/docs/mt5/platform/img/payments_nomupay_currency.png "Specify the payment currency")

## Setting up callback requests [#](payments_nomupay#callback)

Callback requests are used to promptly receive payment statuses. The use of callbacks is optional. However, if you do not configure them, updating payment statuses may take longer. This could negatively impact the quality of your customer service.

To ensure proper generation of the addresses, you need to configure the platform accordingly. On the platform side, the callbacks are received and processed by access servers. To configure notifications:

1\. Register a domain (or subdomain for your existing domain). Nomupay will use this address to communicate with the platform, and the endpoint address for callback requests will be formed relative to it.

2\. Associate the domain with the [public IP address](/en/docs/mt5/platform/administration/admin_network/network_add_edit#public) of one or more access servers. For further information, please see [Integration\\Web Services](/en/docs/mt5/platform/administration/integration/integration_web_services).

3\. [Add an SSL certificate](/en/docs/mt5/platform/administration/integration/integration_web_services#ssl) for the specified domain in the Integration\\Web Services section. A certificate issued by a trusted certification authority is mandatory for operation; self-signed certificates are not allowed even for testing environments. Operation is only possible via the HTTPS protocol. HTTP is not supported.

4\. Choose an endpoint address for receiving callback requests. The address is specified relative to the previously selected domain, taking into account the following rules:

<table class="help" cellspacing="0" cellpadding="5" border="0" style="width:100%; border:none; border-spacing:0;"><tbody><tr><td style="vertical-align:top; padding:5px; border:none"><p class="p_CodeExampleWrap"><span class="f_CodeExampleWrap" style="background-color: #b8cde5;">https://broker.com</span><span class="f_CodeExampleWrap" style="background-color: #f3dcdb;">/api/callback/payments/</span><span class="f_CodeExampleWrap" style="background-color: #d7e4bc;">nomupay</span></p></td></tr></tbody></table>

-   The first part is an example of a domain name.
-   The second part is a predefined part of the address that is used for all callback requests related to payment systems. For example, callbacks to https://broker.com/api/callback/my/callback will not reach wallets.
-   The third part is defined by you. You can enter any address, however we strongly recommend using a logical and clear naming system.

5\. Add the selected URL to the allowed list in the Web Services section and specify the list of IP addresses, from which the payment provider will send transaction status notifications. Please contact Nomupay for this list of addresses.

![Add the address for callback requests to the allowed list](/en/docs/mt5/platform/img/payments_nomupay_webservices.png "Add the address for callback requests to the allowed list")

6\. Provide the full Callback URL, including the domain name, to Nomupay staff. They will configure your channel accordingly and provide you with a special password — the Secret Key. This key will be used by the platform to decrypt notifications.

7\. Specify the callback address in the Callback URL parameter in the payment gateway settings. The system will use it to attribute the received request to the relevant wallets. The address must be specified without the domain name. In the Secret Key parameters, enter the previously received password.

![Specify the callback address in the 'Callback URL' parameter](/en/docs/mt5/platform/img/payments_nomupay_parameters.png "Specify the callback address in the 'Callback URL' parameter")

<table class="container" cellspacing="0" cellpadding="0" border="0"><tbody><tr class="container"><td class="container" colspan="2" style="width:50%;"><h1>Extended Authentication</h1><p class="p_txt"><span class="f_txt">The trading platform provides the option of advanced authentication using SSL certificates, which greatly increases the system security. Extended authentication can be enabled on the trade server. When it is enabled, <a href="/en/docs/mt5/iphone/settings_accounts/account_connect" class="topiclink">the standard authentication</a> is still active. In any case, users need to enter their account details.</span></p><div><div class="parent-table"><table class="attentable" cellspacing="0" cellpadding="8" border="1"><tbody><tr class="attentable"><td class="attentable"><ul><li class="p_tableatten"><span class="f_tableatten">The authentication algorithm is generally accepted and secure. It is fully analogous to the SSL authentication.</span></li><li class="p_tableatten"><span class="f_tableatten">Connection between the client and server is held over its own protocol with the encryption of all data transmitted.</span></li><li class="p_tableatten"><span class="f_tableatten">A public key can be freely distributed and used to authenticate the message, which is signed using a secret key. It is guaranteed that knowing the public key, it is impossible to compute the secret key within a reasonable time. Calculation of the secret key based on the public one, even on powerful up-to-date computers, can take tens or hundreds of years.</span></li></ul><ul><li class="p_tableatten"><span class="f_tableatten">The certificate is not required when connecting using an investor password.</span></li></ul></td></tr></tbody></table></div></div><h2><a name="generation" class="hmanchor"></a>Order of Generating and Receiving a Certificate <a class="anchor" href="extended_authorization#generation">#</a></h2><p class="p_txt"><span class="f_txt">When trying to login using an account with the extended authentication, you will need to go through <a href="/en/docs/mt5/iphone/settings_accounts/account_connect" class="topiclink">standard authentication</a>. After that, the trade server sends a request to the trading platform to generate two keys: private and public. The public key is sent to the trade server.</span></p><p class="p_txt"><span class="f_txt">Based on the account details, the server generates a certificate and signs it with its private key (the server's private key signature guarantees that the certificate can't be falsified).</span></p><p class="p_txt"><span class="f_txt">Extended authentication certificate can be obtained via the mobile or desktop version of the platform when opening an account. It can also be generated by a broker and submitted to a trader.</span></p></td></tr><tr class="container"><td class="container" style="width:50%;"><h2><a name="mobile_side" class="hmanchor"></a>Certificate Installation when Opening Account in the Mobile Platform or through a Broker <a class="anchor" href="extended_authorization#mobile_side">#</a></h2><p class="p_txt"><span class="f_txt">When <a href="/en/docs/mt5/iphone/settings_accounts/account_open" class="topiclink">opening a demo account via the mobile platform</a> on the server with the enabled extended authentication mode, the certificate will be generated and automatically installed on a mobile device. Upon connection using the new account, the installed certificate can be viewed in <a href="/en/docs/mt5/iphone/settings/settings_certificates" class="topiclink">"Certificates"</a> section of MetaTrader 5 iPhone settings.</span></p><p class="p_txt"><span class="f_txt">Similarly, the certificate is generated on the broker's side. After receiving the account number and password, the trader goes through the primary basic authorization on the mobile device. After that the certificate is generated.</span></p><p class="p_txt"><span class="f_txt">No additional operations with the certificate are required.</span></p></td><td class="container" style="width:50%;"><p class="p_Image"><img class="help" alt="Installed certificates" title="Installed certificates" width="324" height="578" src="/en/docs/mt5/iphone/img/certificates.png"></p></td></tr><tr class="container"><td class="container" colspan="2" style="width:50%;"><div><div class="parent-table"><table class="attentable" cellspacing="0" cellpadding="8" border="1"><tbody><tr class="attentable"><td class="attentable"><ul><li class="p_tableatten"><span class="f_tableatten">The certificate is only generated if the trader has not generated any certificate for this account (for example, by connecting to it through a desktop platform).</span></li><li class="p_tableatten"><span class="f_tableatten">If the certificate has been generated on the mobile device, it is impossible to export the certificate for authentication from a desktop platform version. If you want to connect to the account by using both the mobile and desktop platform, contact your broker in order to delete the previously generated certificate. Then request a new certificate from the desktop platform and import it to your mobile device following <a href="/en/docs/mt5/iphone/settings_accounts/extended_authorization#terminal_side" class="topiclink">instructions</a> below.</span></li></ul></td></tr></tbody></table></div></div></td></tr><tr class="container"><td class="container" colspan="2" style="width:50%;"><h2><a name="terminal_side" class="hmanchor"></a>Certificate Installing When Opening an Account in the Desktop Platform <a class="anchor" href="extended_authorization#terminal_side">#</a></h2><p class="p_txt"><span class="f_txt">When <a href="https://www.metatrader5.com/en/terminal/help/startworking/acc_open" target="_blank" class="weblink" title="MetaTrader 5: Demo Account Opening">opening a demo account in the desktop platform</a>, the extended authentication certificate is also generated. A trader needs to specify the passwords for the certificate protection before receiving it.</span></p><p class="p_Image"><img class="help" alt="Certificate password" title="Certificate password" width="511" height="275" src="/en/docs/mt5/iphone/img/certificate_password_client.png"></p><p class="p_txt"><span class="f_txt">This window contains the following fields and setting:</span></p><ul><li class="p_ol"><a name="password" class="hmanchor"></a><span class="f_ol" style="font-weight: bold;">Password</span><span class="f_ol"> — a password for the certificate installation;</span></li><li class="p_ol"><span class="f_ol" style="font-weight: bold;">Confirm password</span><span class="f_ol"> — confirmation of the password to avoid mistyping;</span></li><li class="p_ol"><span class="f_ol" style="font-weight: bold;">Add the certificate to the Windows storage</span><span class="f_ol"> — if this option is ticked off, the certificate will be automatically installed to the operating system storage.</span></li></ul><div><div class="parent-table"><table class="attentable" cellspacing="0" cellpadding="8" border="1"><tbody><tr class="attentable"><td class="attentable"><p class="p_tableatten"><span class="f_tableatten">Password set to the certificate must contain at least two types of characters (lower case letters, upper case letters, numbers or special characters) and consist of not less than five characters.</span></p></td></tr></tbody></table></div></div><p class="p_txt"><span class="f_txt">After all of the required data are specified, tap "Continue". After that the certificate is packed and protected by the specified password. The resulting *.pfx file of the certificate is saved in /platform_folder/config/certificates of the trading platform to enable its further transfer. Names to the certificate files are assigned according to the following rule: </span><span class="f_txt" style="font-style: italic;">Login_ID_Name.pfx</span><span class="f_txt">, where:</span></p><ul><li class="p_ol"><span class="f_ol" style="font-weight: bold;">Login</span><span class="f_ol"> is the account number;</span></li><li class="p_ol"><span class="f_ol" style="font-weight: bold;">ID</span><span class="f_ol"> is a short name of the company the account was opened in;</span></li><li class="p_ol"><span class="f_ol" style="font-weight: bold;">Name</span><span class="f_ol"> is the name of a client specified when creating the account.</span></li></ul><div><div class="parent-table"><table class="attentable" cellspacing="0" cellpadding="8" border="1"><tbody><tr class="attentable"><td class="attentable" style="vertical-align:middle;"><ul><li class="p_tableatten"><span class="f_tableatten">Even having access to the *.pfx file, the certificate cannot be used without the password.</span></li><li class="p_tableatten"><span class="f_tableatten">Certificates are generated only during the first account connection or when a certificate is intentionally reset on the server.</span></li></ul></td></tr></tbody></table></div></div><p class="p_txt"><span class="f_txt">In order to use the certificate in the MetaTrader 5 for iPhone, you should transfer it to the mobile platform.</span></p><h3>Certificate Transfer Procedure</h3><p class="p_txt"><span class="f_txt">A certificate can be securely transferred via a trade server:</span></p><ul><li class="p_ol"><span class="f_ol">The certificate is first encrypted on the desktop platform: the account owner sets the password for certificate encryption using the secure AES-256 algorithm. This password is only know to the user, while it is not sent to the server.</span></li><li class="p_ol"><span class="f_ol">Further, the encrypted certificate is sent to the trade server, where it will be stored until it is received through the mobile platform, but no more than one hour.</span></li><li class="p_ol"><span class="f_ol">To receive the certificate on a mobile device, the user must connect to the trading account from the mobile platform. After connecting, the user is prompted to import the certificate. To proceed with the import, the user needs to specify the password that was used for the certificate encryption on the desktop platform.</span></li></ul><p class="p_txt"><span class="f_txt">Certificate transfer process is secure: the trade server is only used as an intermediate storage, while the certificate is encrypted on the user's side. The certificate password is not transmitted to or stored on the trade server.</span></p><h3>How to Transfer a Certificate</h3><p class="p_txt"><span class="f_txt">Connect to your account from the desktop platform and select "Transfer Certificate" in its context menu:</span></p><p class="p_Image"><img class="help" alt="How to transfer a certificate to a mobile device" title="How to transfer a certificate to a mobile device" width="750" height="631" src="/en/docs/mt5/iphone/img/transfer_certificate.png"></p><p class="p_txt"><span class="f_txt">Enter the master password of the account to confirm that it is yours. Next, set a password to protect the certificate before sending it to the server, or use a random password that is generated automatically. Set a password that has at least 8 digits.</span></p><p class="p_txt"><span class="f_txt">After successfully sending the certificate to the server, open the mobile platform and connect to the account. You will be immediately prompted to import the certificate. Confirm and enter the password that you have set from the desktop platform.</span></p><p class="p_Image"><img class="help" alt="How to import a certificate to a mobile device" title="How to import a certificate to a mobile device" width="650" height="558" src="/en/docs/mt5/iphone/img/transfer_certificate_import.png"></p><p class="p_txt"><span class="f_txt">You can view the imported certificate in the "About — Certificates" section.</span></p></td></tr><tr class="container"><td class="container" style="width:50%;"><p class="p_Image"><img class="help" alt="Importing a certificate from a PFX file" title="Importing a certificate from a PFX file" width="324" height="558" src="/en/docs/mt5/iphone/img/certificate_import.png"></p></td><td class="container" style="width:50%;"><h3>Another Transfer Option</h3><p class="p_txt"><span class="f_txt">You can transfer the certificate manually. Send a PFX file from your desktop computer to a mobile device in any preferred way, such as email, messenger, etc. Save it to the Files app on your device. Next, connect to your account in the mobile platform and tap "Import Certificate" in the dialog. Select the previously saved file and enter the certificate protection password.</span></p></td></tr><tr class="container"><td class="container" style="width:50%;"><h3>Not Installed Certificates</h3><p class="p_txt"><span class="f_txt">When launching MetaTrader 5 iPhone, you will see the list of certificates to be installed or deleted (downloaded via iTunes). This list will appear every time you launch the application until you install or remove each of the certificates.</span></p></td><td class="container" style="width:50%;"><p class="p_Image"><img class="help" alt="Not installed certificates" title="Not installed certificates" width="322" height="553" src="/en/docs/mt5/iphone/img/notinstalled_certificates.png"></p></td></tr><tr class="container"><td class="container" style="width:50%;"><p class="p_Image"><img class="help" alt="Certificate installation password" title="Certificate installation password" width="322" height="554" src="/en/docs/mt5/iphone/img/certificate_install_password.png"></p></td><td class="container" style="width:50%;"><h3>Certificate Installation</h3><p class="p_txt"><span class="f_txt">Tap on a certificate to install it. Enter <a href="/en/docs/mt5/iphone/settings_accounts/extended_authorization#password" class="topiclink">certificate password</a> that was specified when obtaining the certificate via the desktop platform.</span></p><p class="p_txt"><span class="f_txt">Tap "Done" to install the certificate. Now you can <a href="/en/docs/mt5/iphone/settings_accounts/account_connect" class="topiclink">connect to your previously opened account</a> in the extended authentication mode. The appropriate certificate will be checked during each connection.</span></p><div><div class="parent-table"><table class="attentable" cellspacing="0" cellpadding="8" border="1"><tbody><tr class="attentable"><td class="attentable"><p class="p_tableatten"><span class="f_tableatten">Installed certificates are managed in <a href="/en/docs/mt5/iphone/settings/settings_certificates" class="topiclink">"Certificates"</a> section of MetaTrader 5 iPhone settings.</span></p></td></tr></tbody></table></div></div></td></tr><tr class="container"><td class="container" style="width:50%;"><h3>Deleting certificate</h3><p class="p_txt"><span class="f_txt">If you have installed a wrong certificate via iTunes, delete it when launching MetaTrader 5 iPhone. To do this, tap "Edit" and then <img class="help" alt="Delete" title="Delete" width="22" height="22" src="/en/docs/mt5/iphone/img/delete_icon.png"> on the left.</span></p></td><td class="container" style="width:50%;"><p class="p_Image"><img class="help" alt="Deleting certificate" title="Deleting certificate" width="322" height="553" src="/en/docs/mt5/iphone/img/delete_notinstalled_certificate.png"></p></td></tr><tr class="container"><td class="container" style="width:50%;"><p class="p_Image"><img class="help" alt="Invalid certificate" title="Invalid certificate" width="322" height="554" src="/en/docs/mt5/iphone/img/incorrect_certificate.png"></p></td><td class="container" style="width:50%;"><p class="p_txt"><a name="broker_side" class="hmanchor"></a><span class="f_H2">Installing a Certificate from the Brokerage Company</span></p><p class="p_txt"><span class="f_ol">Real accounts (not demo ones) can be opened by a brokerage company. The extended authentication mode can also be used for such accounts.</span></p><p class="p_txt"><span class="f_ol">In that case a company submits pfx certificate file together with the rest of the data necessary for authorization. This certificate must be <a href="/en/docs/mt5/iphone/settings_accounts/extended_authorization#terminal_side" class="topiclink">installed</a> via iTunes according to the instructions received in the desktop version of the platform.</span></p><h2><a name="no_certificate" class="hmanchor"></a>No Certificate <a class="anchor" href="extended_authorization#no_certificate">#</a></h2><p class="p_txt"><span class="f_txt">If the extended authentication mode is used for an account but the appropriate certificate has not been installed, an error message will appear when when try to connect to your account.</span></p><p class="p_txt"><span class="f_txt">Install the appropriate certificate following the instructions above to continue working with your account.</span></p></td></tr></tbody></table>

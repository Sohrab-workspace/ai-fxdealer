<table class="container" cellspacing="0" cellpadding="0" border="0"><tbody><tr class="container"><td class="container" colspan="2" style="width:50%;"><h1>Extended Authentication</h1><p class="p_txt"><span class="f_txt">The trading platform provides an option of extended authorization using SSL certificates, which greatly increases the system security. The extended authentication can be enabled on the server. When it is enabled, <a href="/en/docs/mt5/android/settings_accounts/account_connect" class="topiclink">the standard authentication</a> is still active. In any case, users need to specify their account details.</span></p><div><div class="parent-table"><table class="attentable" cellspacing="0" cellpadding="8" border="1"><tbody><tr class="attentable"><td class="attentable"><ul><li class="p_tableatten"><span class="f_tableatten">The authentication algorithm is standard and highly reliable; it is completely similar to SSL authentication.</span></li><li class="p_tableatten"><span class="f_tableatten">Connection between the client and server is established over a custom protocol, while any data transmitted is securely encrypted.</span></li><li class="p_tableatten"><span class="f_tableatten">A public key can be freely distributed and used to authenticate the message, which is signed using a secret key. It is guaranteed, that knowing a public key, it is impossible to compute the secret key within a reasonable time. The calculation of a secret key based on a public one, even on powerful modern computers, can take tens or even hundreds of years.</span></li></ul><ul><li class="p_tableatten"><span class="f_tableatten">The certificate is not required when connecting using an investor password.</span></li></ul></td></tr></tbody></table></div></div><h2><a name="generation" class="hmanchor"></a>Order of Generating and Receiving a Certificate <a class="anchor" href="extended_authorization#generation">#</a></h2><p class="p_txt"><span class="f_txt">When trying to authorize using an account with the extended authorization enabled, you will need to go through <a href="/en/docs/mt5/android/settings_accounts/account_connect" class="topiclink">standard authorization</a>. After that, the trade server sends a request to the platform to generate two keys: private and public. The public key is sent to the trade server.</span></p><p class="p_txt"><span class="f_txt">Based on the account data, the server generates a certificate and signs it with its private key (the server's private key signature guarantees that the certificate cannot be falsified).</span></p><p class="p_txt"><span class="f_txt">Extended authentication certificate can be obtained via the mobile or desktop version of the platform when opening an account. It can also be generated by a broker and submitted to a trader.</span></p></td></tr><tr class="container"><td class="container" style="width:50%;"><h2><a name="mobile_side" class="hmanchor"></a>How to Install the Certificate when Opening an Account in the Mobile Platform <a class="anchor" href="extended_authorization#mobile_side">#</a></h2><p class="p_txt"><span class="f_txt">When <a href="/en/docs/mt5/android/settings_accounts/account_open" class="topiclink">opening a demo account via the mobile platform</a> on the server with the enabled extended authentication mode, the certificate will be generated and automatically installed on a mobile device.</span></p><p class="p_txt"><span class="f_txt">Similarly, the certificate is generated on the broker's side. After receiving the account number and password, the trader goes through the primary basic authorization on the mobile device. After that the certificate is generated.</span></p><div><div class="parent-table"><table class="attentable" cellspacing="0" cellpadding="8" border="1"><tbody><tr class="attentable"><td class="attentable"><p class="p_tableatten"><span class="f_tableatten">The certificate is only generated if the trader has not generated a certificate for this account (for example, by connecting to the account from a desktop platform).</span></p></td></tr></tbody></table></div></div><p class="p_txt"><span class="f_txt">&nbsp;</span></p></td><td class="container" style="width:50%;"><p class="p_Image"><img class="help" alt="Certificate password" title="Certificate password" width="326" height="512" src="/en/docs/mt5/android/img/certificate_password.png"></p></td></tr><tr class="container"><td class="container" colspan="2" style="width:50%;"><p class="p_txt"><span class="f_txt">The certificate file is saved on the device during installation. To protect the certificate, you need to specify the password and confirm it.</span></p><div><div class="parent-table"><table class="attentable" cellspacing="0" cellpadding="8" border="1"><tbody><tr class="attentable"><td class="attentable"><p class="p_tableatten"><span class="f_tableatten">The password set to the certificate must contain at least two types of characters (lower case letters, upper case letters, numbers or special characters), and consist of not less than five characters.</span></p></td></tr></tbody></table></div></div><p class="p_txt"><span class="f_txt">After all of the required data are specified, press "Next". The certificate will be generated and saved in protected storage of the device internal memory. Only MetaTrader 5 Android application will have access to the storage.</span></p><p class="p_txt"><span class="f_txt">After connecting using the new account, the installed certificate can be viewed in <a href="/en/docs/mt5/android/settings_accounts" class="topiclink">"Accounts"</a> section. No additional operations with the certificate are required.</span></p><h2><a name="terminal_side" class="hmanchor"></a>How to Install a Certificate When Opening an Account from the Desktop Platform <a class="anchor" href="extended_authorization#terminal_side">#</a></h2><p class="p_txt"><span class="f_txt">When <a href="https://www.metatrader5.com/en/terminal/help/startworking/acc_open" target="_blank" class="weblink" title="MetaTrader 5: Demo Account Opening">opening a demo account from the desktop version of the trading platform,</a> the extended verification certificate is also generated. A trader needs to specify the passwords for the certificate protection before receiving the certificate.</span></p><p class="p_Image"><img class="help" alt="Certificate password" title="Certificate password" width="511" height="275" src="/en/docs/mt5/android/img/certificate_password_client.png"></p><p class="p_txt"><span class="f_txt">This window contains the following fields and setting:</span></p><ul><li class="p_ol"><a name="password" class="hmanchor"></a><span class="f_ol" style="font-weight: bold;">Password</span><span class="f_ol"> — a password for the certificate installation;</span></li><li class="p_ol"><span class="f_ol" style="font-weight: bold;">Confirm password</span><span class="f_ol"> — confirmation of the password to avoid mistyping;</span></li><li class="p_ol"><span class="f_ol" style="font-weight: bold;">Add the certificate top the Windows storage</span><span class="f_ol"> — if this option is ticked off, the certificate will be automatically installed to the operating system storage.</span></li></ul><div><div class="parent-table"><table class="attentable" cellspacing="0" cellpadding="8" border="1"><tbody><tr class="attentable"><td class="attentable"><p class="p_tableatten"><span class="f_tableatten">The password set to the certificate must contain at least two types of characters (lower case letters, upper case letters, numbers or special characters), and consist of not less than five characters.</span></p></td></tr></tbody></table></div></div><p class="p_txt"><span class="f_txt">After all of the required data are specified, press "Continue". After that the certificate is packed and protected by the specified password. The resulting certificate file *.pfx is stored in /platform_folder/config/certificates, from which it can be transferred later. Names to the certificate files are assigned according to the following rule: </span><span class="f_txt" style="font-style: italic;">Login_ID_Name.pfx</span><span class="f_txt">, where:</span></p><ul><li class="p_ol"><span class="f_ol" style="font-weight: bold;">Login</span><span class="f_ol"> — account number;</span></li><li class="p_ol"><span class="f_ol" style="font-weight: bold;">ID</span><span class="f_ol"> — short name of the company the account was opened in;</span></li><li class="p_ol"><span class="f_ol" style="font-weight: bold;">Name</span><span class="f_ol"> — name of a client specified when creating the account.</span></li></ul><div><div class="parent-table"><table class="attentable" cellspacing="0" cellpadding="8" border="1"><tbody><tr class="attentable"><td class="attentable" style="vertical-align:middle;"><ul><li class="p_tableatten"><span class="f_tableatten">Even having access to the *.pfx file, the certificate cannot be used without the password.</span></li><li class="p_tableatten"><span class="f_tableatten">Certificates are generated only during the first account connection or when a certificate is intentionally reset on the server.</span></li></ul></td></tr></tbody></table></div></div><p class="p_txt"><span class="f_txt">In order to use the certificate in the MetaTrader 5 for Android, you should transfer it to the mobile platform.</span></p><h3>Certificate transfer</h3><p class="p_txt"><span class="f_txt">The certificate is transferred securely via a trading server:</span></p><ul><li class="p_ol"><span class="f_ol">The certificate is first encrypted on the desktop platform: the account owner sets the password for certificate encryption using the secure AES-256 algorithm. This password is only known to the user, and it will not be sent to the server.</span></li><li class="p_ol"><span class="f_ol">Next, the encrypted certificate is sent to the trade server where it is stored before receipt via a mobile platform but no more than an hour.</span></li><li class="p_ol"><span class="f_ol">In order to receive the certificate, the user should connect to the account via the mobile platform. After connecting, the user is prompted to import the certificate. To do this, they should enter the password that was used to encrypt the certificate in the desktop platform.</span></li></ul><p class="p_txt"><span class="f_txt">The certificate transfer is secure: the trading server is used solely as an intermediate storage, while encryption is performed at the user's side. The certificate password is not transferred or stored on the trade server.</span></p><h3>How to transfer the certificate</h3><p class="p_txt"><span class="f_txt">Connect to the account via the desktop platform and select "Transfer SSL certificate to mobile device" in its context menu:</span></p><p class="p_Image"><img class="help" alt="Transferring the certificate to a mobile device" title="Transferring the certificate to a mobile device" width="750" height="631" src="/en/docs/mt5/android/img/transfer_certificate.png"></p><p class="p_txt"><span class="f_txt">Specify the master password of the account to confirm that it belongs to you. Next, set a password to protect the certificate before sending it to the server, or use a random password that is generated automatically. The password should consist of at least 8 digits.</span></p><p class="p_txt"><span class="f_txt">After successfully sending the certificate to the server, open the mobile platform and connect to the account. You will immediately be prompted to import the certificate. Confirm and enter the password that you have set from the desktop platform.</span></p><p class="p_Image"><img class="help" alt="How to import a certificate to a mobile device" title="How to import a certificate to a mobile device" width="607" height="512" src="/en/docs/mt5/android/img/transfer_certificate_import.png"></p><p class="p_txt"><span class="f_txt">You can view the certificate in Accounts  Certificates.</span></p></td></tr><tr class="container"><td class="container" style="width:50%;"><p class="p_Image"><img class="help" alt="Not installed certificates" title="Not installed certificates" width="326" height="512" src="/en/docs/mt5/android/img/notinstalled_certificates.png"></p></td><td class="container" style="width:50%;"><h3>Another transfer option</h3><p class="p_txt"><span class="f_txt">If you want to use the certificate in MetaTrader 5 Android, choose one of the following methods to copy it to your device:</span></p><ul><li class="p_ol"><span class="f_ol">connect your device to a PC and copy the file;</span></li><li class="p_ol"><span class="f_ol">send your certificate via e-mail and open the appropriate message on the device;</span></li><li class="p_ol"><span class="f_ol">send a certificate to your device via bluetooth.</span></li></ul><p class="p_txt"><span class="f_txt">Launch MetaTrader 5 Android and move to "Accounts" section. Press <img class="help" alt="Certificates button" title="Certificates button" width="20" height="20" src="/en/docs/mt5/android/img/certificates_button.png"> to open the list of the already installed certificates. Next, press <img class="help" alt="Open an Account" title="Open an Account" width="20" height="20" src="/en/docs/mt5/android/img/symbol_add_button.png"> to search for the certificate files on the device.</span></p><p class="p_txt"><span class="f_txt">Tap on a certificate to install it. Enter <a href="/en/docs/mt5/android/settings_accounts/extended_authorization#password" class="topiclink">certificate password</a> that was specified when obtaining the certificate via the desktop version of the trading platform.</span></p></td></tr><tr class="container"><td class="container" colspan="2" style="width:50%;"><p class="p_Image"><img class="help" alt="Certificate installation password" title="Certificate installation password" width="326" height="512" src="/en/docs/mt5/android/img/certificate_install_password.png"><span class="f_Image"> <img class="help" alt="Certificate import confirmation" title="Certificate import confirmation" width="326" height="512" src="/en/docs/mt5/android/img/certificate_install_confirm.png"></span></p><p class="p_txt"><span class="f_txt">Tap "OK" after installation. Certificate data window will appear. Check the data and confirm installation:</span></p><p class="p_txt"><span class="f_txt">Now you can <a href="/en/docs/mt5/android/settings_accounts/account_connect" class="topiclink">connect to your previously opened account</a> in the extended authorization mode. The appropriate certificate will be checked during each connection.</span></p><div><div class="parent-table"><table class="attentable" cellspacing="0" cellpadding="8" border="1"><tbody><tr class="attentable"><td class="attentable"><p class="p_tableatten"><span class="f_tableatten">Installed certificates are managed in "Accounts" section.</span></p></td></tr></tbody></table></div></div></td></tr><tr class="container"><td class="container" style="width:50%;"><h2>Deleting certificate</h2><p class="p_txt"><span class="f_txt">If you have installed a wrong certificate, it can be deleted. Press and hold the certificate. The context menu will appear:</span></p><p class="p_Image"><span class="f_Image">&nbsp;</span></p><p class="p_txt"><span class="f_txt">Tap "Delete certificate".</span></p></td><td class="container" style="width:50%;"><p class="p_Image"><img class="help" alt="Certificate context menu" title="Certificate context menu" width="326" height="512" src="/en/docs/mt5/android/img/certificate_context.png"></p></td></tr><tr class="container"><td class="container" colspan="2" style="width:50%;"><p class="p_txt"><a name="broker_side" class="hmanchor"></a><span class="f_H2">Installing a Certificate from the Brokerage Company</span></p><p class="p_txt"><span class="f_ol">Real accounts (not demo ones) can be opened by a brokerage company. The extended authorization mode can also be used for such accounts.</span></p><p class="p_txt"><span class="f_ol">In that case a company submits pfx certificate file together with the rest of the data necessary for authentication. This certificate must be <a href="/en/docs/mt5/android/settings_accounts/extended_authorization#terminal_side" class="topiclink">installed</a> on the device according to the instructions received via the desktop version of the trading platform.</span></p></td></tr><tr class="container"><td class="container" style="width:50%;"><p class="p_Image"><img class="help" alt="Invalid certificate" title="Invalid certificate" width="326" height="512" src="/en/docs/mt5/android/img/incorrect_certificate.png"></p></td><td class="container" style="width:50%;"><h2><a name="no_certificate" class="hmanchor"></a>Certificate Absence <a class="anchor" href="extended_authorization#no_certificate">#</a></h2><p class="p_txt"><span class="f_txt">If the extended authentication mode is used for an account but the appropriate certificate has not been installed, an error message will appear when you try to connect to your account.</span></p><p class="p_txt"><span class="f_txt">Install the appropriate certificate following the instructions above to continue working with your account.</span></p></td></tr><tr class="container"><td class="container" style="width:50%;"><h2><a name="export" class="hmanchor"></a>Certificate Export <a class="anchor" href="extended_authorization#export">#</a></h2><p class="p_txt"><span class="f_txt">The previously installed certificate can be exported to a file. This will allow using it on other devices or the desktop version of the trading platform.</span></p><p class="p_txt"><span class="f_txt">Move to "Accounts" section. Long tap on the certificate to open its context menu.</span></p><p class="p_txt"><span class="f_txt">Then tap "Export certificate" and enter the password used to protect the certificate during <a href="/en/docs/mt5/android/settings_accounts/extended_authorization#generation" class="topiclink">installation</a>.</span></p></td><td class="container" style="width:50%;"><p class="p_Image"><img class="help" alt="Certificate context menu" title="Certificate context menu" width="326" height="512" src="/en/docs/mt5/android/img/certificate_context.png"></p></td></tr><tr class="container"><td class="container" colspan="2" style="width:50%;"><p class="p_txt"><span class="f_txt">Enter the password and press "OK". The certificate will be exported to the external storage of the device. The name of the generated file will be displayed in the new window:</span></p><p class="p_Image"><img class="help" alt="Password for certificate export" title="Password for certificate export" width="326" height="512" src="/en/docs/mt5/android/img/certificate_export_password.png"><span class="f_Image"> <img class="help" alt="Certificate exported" title="Certificate exported" width="326" height="512" src="/en/docs/mt5/android/img/certificate_exported.png"></span></p><p class="p_txt"><span class="f_txt">Now, you can copy the certificate from the device by connecting it to PC, via Bluetooth, e-mail, etc.</span></p></td></tr></tbody></table>

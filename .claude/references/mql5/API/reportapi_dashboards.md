# Dashboards

Dashboards form a separate type of reports that allow combining different data and their presentation on one sheet.

A dashboard consists of widgets representing data either in the form of HTML content, or in the form of a diagram and/or table. Each of these cases uses its own source data.

-   [IMTDataset](/en/docs/mt5/api/reportapi_dataset/imtdataset) — set of tabular data for widgets displaying diagrams and tables. Working with these data is completely equivalent to [tabular reports](/en/docs/mt5/api/reportapi_tables).
-   [IMTReportDashboardHtml](/en/docs/mt5/api/reportapi_dashboard/imtreportdashboardhtml) - data set for widgets displaying HTML content. Working with these data is completely equivalent to [HTML reports](/en/docs/mt5/api/reportapi_html).

A data set is bound to the widget using the [IMTReportDashboardWidget::Html](/en/docs/mt5/api/reportapi_dashboard/imtreportdashboardwidget/imtreportdashboardwidget_html) or [IMTReportDashboardWidget::Data](/en/docs/mt5/api/reportapi_dashboard/imtreportdashboardwidget/imtreportdashboardwidget_data) method depending on its type. The same data set can be used in different widgets, for example for displaying the same data in different ways or displaying different parts of the data set in separate tables.

Procedure:

-   The column is cleared using the [IMTDatasetColumn::Clear](/en/docs/mt5/api/reportapi_dataset/imtdatasetcolumn/imtdatasetcolumn_clear) method;
-   The "Name" is assigned to the column using the [IMTDatasetColumn::Name](/en/docs/mt5/api/reportapi_dataset/imtdatasetcolumn/imtdatasetcolumn_name) method;
-   ID "1" is assigned to the column using the [IMTDatasetColumn::ColumnID](/en/docs/mt5/api/reportapi_dataset/imtdatasetcolumn/imtdatasetcolumn_columnid) method;
-   A shift in bytes is assigned to the column using the [IMTDatasetColumn::Offset](/en/docs/mt5/api/reportapi_dataset/imtdatasetcolumn/imtdatasetcolumn_offset) method. "Offsetof" macros is used in the example to avoid calculation of a shift for each column (calculated as previous columns size). The stddef.h file should be included into the project to use the macros. The name of a field from the previously described table record structure ([TableRecord](/en/docs/mt5/api/reportapi_tables#table_record)) is transferred to the macros.
-   "String" type is assigned to the column using the [IMTDatasetColumn::Type](/en/docs/mt5/api/reportapi_dataset/imtdatasetcolumn/imtdatasetcolumn_type) method;
-   A size in bytes is assigned to the column using the [IMTDatasetColumn::Size](/en/docs/mt5/api/reportapi_dataset/imtdatasetcolumn/imtdatasetcolumn_size) method. The MtFieldSize macros is used to simplify the defining of a column size. The name of a field from the previously described table record structure (TableRecord) is also transferred to the macro. This macro is not standard. Its realization should be added to the stdafx.h file.
-   The generated column is then added to the IMTReportAPI object copy with the help of the [IMTDataset::ColumnAdd](/en/docs/mt5/api/reportapi_dataset/imtdataset/imtdataset_columnadd) method.
-   In case of an adding error, the column object should be necessarily freed by using the Release method ([IMTDatasetColumn::Release](/en/docs/mt5/api/reportapi_dataset/imtdatasetcolumn/imtdatasetcolumn_release)).

Procedure:

-   Whether the list of logins is received is checked in the 'if' statement.
-   A client record object is then created by using the [IMTReportAPI::UserCreate](/en/docs/mt5/api/imtreportapi/imtreportapi_user/imtreportapi_usercreate) method. In case of a creation error, received list of logins is freed by using the [IMTReportAPI::Free](/en/docs/mt5/api/imtreportapi/imtreportapi_common/imtreportapi_free) method.
-   Client records (IMTUser) for each of the received logins are requested in the 'for' loop using the [IMTReportAPI::UserGet](/en/docs/mt5/api/imtreportapi/imtreportapi_user/imtreportapi_userget) method.
-   The 'record' table record structure ([TableRecord](/en/docs/mt5/api/reportapi_tables#table_record)) is set to zero.
-   A name from a client record is copied to an appropriate record structure field using the auxiliary CMTStr::Copy method.
-   A leverage value is copied from a client record to an appropriate record structure field.
-   Received row is assigned to the api report object using the [IMTReportAPI::TableRowWrite](/en/docs/mt5/api/imtreportapi/imtreportapi_table/imtreportapi_table_row/imtreportapi_tablerowwrite) method. In case of a record error, received list of logins is freed by using the [IMTReportAPI::Free](/en/docs/mt5/api/imtreportapi/imtreportapi_common/imtreportapi_free) method.
-   After the report generation is complete, the previously received array of logins is freed using the [IMTReportAPI::Free](/en/docs/mt5/api/imtreportapi/imtreportapi_common/imtreportapi_free) method.
-   The client record object is freed by the Release method ([IMTUser::Release](/en/docs/mt5/api/reference_user/imtuser/imtuser_release)).

# Trading Functions

This section contains descriptions of functions, using which you can implement trading activities from an application developed using the MetaTrader 5 Manager API.

Trading functions are divided into the following sections:

-   [Dealing](/en/docs/mt5/api/imtmanagerapi/imtmanagerapi_trade_operations/imtmanagerapi_dealing) — dealer activity functions.
-   [Trading requests](/en/docs/mt5/api/imtmanagerapi/imtmanagerapi_trade_operations/imtmanagerapi_trading_request) — functions for working with trading requests queue.
-   [Auxiliary Functions](/en/docs/mt5/api/imtmanagerapi/imtmanagerapi_trade_operations/imtmanagerapi_trade_auxiliary) — functions for calculating profit, margin requirements and conversion rates.

## Dealer and Supervisor Modes [#](imtmanagerapi_trade_operations#dealer_supervisor)

Depending on the [rights](/en/docs/mt5/api/config_manager/imtconmanager/imtconmanager_enum#enmanagerrights) assigned to an account, a manager can perform the following activities related to trade operations on the server:

### Dealing

The Dealing permission ([IMTConManager:RIGHT\_TRADES\_DEALER](/en/docs/mt5/api/config_manager/imtconmanager/imtconmanager_enum#enmanagerrights)) allows a manager to handle trade requests received in accordance with the [routing rules](/en/docs/mt5/api/config_route). Also, this right allows the manager the perform trade operations.

The dealing procedure:

-   After connecting to a server using the [IMTManagerAPI::Connect](/en/docs/mt5/api/imtmanagerapi/imtmanagerapi_connection/imtmanagerapi_connect) method with the required [pumping modes](/en/docs/mt5/api/imtmanagerapi/imtmanagerapi_connection/imtmanagerapi_enpumpmodes) (for orders, positions, accounts, groups and symbols) and starting dealing using the [IMTManagerAPI:DealerStart](/en/docs/mt5/api/imtmanagerapi/imtmanagerapi_trade_operations/imtmanagerapi_dealing/imtmanagerapi_dealerstart) method, the manager receives the status of the queue of requests forwarded to the manager in accordance with the routing rules. Next, it will receive the changes of the queue (added, edited and deleted requests) using the methods of the [IMTRequestSink](/en/docs/mt5/api/reference_trading/trading_request/imtrequestsink) interface.
-   To capture a request from a queue for processing, methods [IMTManagerAPI:DealerGet](/en/docs/mt5/api/imtmanagerapi/imtmanagerapi_trade_operations/imtmanagerapi_dealing/imtmanagerapi_dealerget) and [IMTManagerAPI:DealerLock](/en/docs/mt5/api/imtmanagerapi/imtmanagerapi_trade_operations/imtmanagerapi_dealing/imtmanagerapi_dealerlock) are used. A captured request must necessarily be processed by the dealer using the [IMTManagerAPI::DealerAnswer](/en/docs/mt5/api/imtmanagerapi/imtmanagerapi_trade_operations/imtmanagerapi_dealing/imtmanagerapi_dealeranswer) method. In this method, the object request confirmation [IMTConfirm](/en/docs/mt5/api/reference_trading/trading_request/imtconfirm) is passed, where the dealer specifies how the request should be executed.
-   Once confirmed by a dealer, the request appears on the queue to be executed by the server. The result of request execution can be received based on its identifier ([IMTRequest::ID](/en/docs/mt5/api/reference_trading/trading_request/imtrequest/imtrequest_id)) using the [IMTRequest::Result\*](/en/docs/mt5/api/reference_trading/trading_request/imtrequest/imtrequest_resultretcode) methods.

### Supervisor

The Supervisor right ([IMTConManager:RIGHT\_TRADES\_SUPERVIOSR](/en/docs/mt5/api/config_manager/imtconmanager/imtconmanager_enum#enmanagerrights)) allows a manager to view the entire queue of requests forwarded to him or her from available client groups, and track the process of request processing by other dealers.

Description of the Supervisor mode:

-   After connecting to a server using the [IMTManagerAPI::Connect](/en/docs/mt5/api/imtmanagerapi/imtmanagerapi_connection/imtmanagerapi_connect) method with the required [pumping modes](/en/docs/mt5/api/imtmanagerapi/imtmanagerapi_connection/imtmanagerapi_enpumpmodes) (for orders, positions, accounts, groups and symbols), the manager receives the status of the queue of requests, which the manager can view in accordance with the client group settings ([IMTConManager::Group\*](/en/docs/mt5/api/config_manager/imtconmanager/imtconmanager_groupadd)).
-   Then the manager will receive changes of the queue (added, edited and deleted requests) using the methods of the [IMTRequestSink](/en/docs/mt5/api/reference_trading/trading_request/imtrequestsink) interface. Also, the manager will be able to get the results of request processing by other dealers using the [IMTRequest::Result\*](/en/docs/mt5/api/reference_trading/trading_request/imtrequest/imtrequest_resultretcode) methods.
-   When switching to the dealing mode using the [IMTManagerAPI:DealerStart](/en/docs/mt5/api/imtmanagerapi/imtmanagerapi_trade_operations/imtmanagerapi_dealing/imtmanagerapi_dealerstart) method, the manager no longer sees the entire queue of requests from client groups available to him or her. Since then, the manager receives only the requests that are forwarded to him or her for processing in accordance with [routing rules](/en/docs/mt5/api/config_route). Thus, the manager simply switches to the dealing mode described above.

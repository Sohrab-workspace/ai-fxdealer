\[in\] [Trade execution object](/en/docs/mt5/api/reference_trading/trading_request/imtexecution).

Return Value

If the command is successfully sent to the execution queue, the [MT\_RET\_OK](/en/docs/mt5/api/retcodes_successful) response code is returned. Otherwise, an error has occurred, which corresponds to the response code.

[MT\_RET\_OK\_NONE](/en/docs/mt5/api/retcodes_successful) code is returned when the gateway is not able to determine the server to which the given trade execution should be sent based on the [IMTExecution type](/en/docs/mt5/api/reference_trading/trading_request/imtexecution/imtexecution_action) and the data passed in it. For example, IMTExecution of the TE\_ORDER\_NEW\_REQUEST type does not include the order ticket on the MetaTrader 5 side.

The [MT\_RET\_ERR\_CONNECTION](/en/docs/mt5/api/retcodes_common) response code is returned if a trade server has not connected to anything yet since launching the gateway. After the first connection, the gateway is able to send executions using this method even if the connection to the trade server has been interrupted. In this case, the messages are sent during the next connection to the trade server.

Note

During the execution of this method the operation is carried out in the trading platform that corresponds to the executed one in the external environment.

The result of applying the [IMTExecution](/en/docs/mt5/api/reference_trading/trading_request/imtexecution) execution object to the server data base can be obtained in the [IMTGatewaySink::OnDealerAnswer](/en/docs/mt5/api/imtgatewaysink/imtgatewaysink_ondealeranswer) method.

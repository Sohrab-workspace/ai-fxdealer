# Data cache

Data cache is used to store data requested from the trading platform database, between report generation times. Use of data cache saves resources when creating reports: there is no need to re-retrieve data from databases and perform related calculations, while the ready data is available in the cache.

Caches are stored on the trade server:

-   In memory, for quick access to frequently used data. The cache is automatically unloaded from the memory if it is not accessed within 2 days.
-   On a hard disk, for long-term access. The cache is automatically deleted from the disk if it is not accessed within 30 days.

## Cache types [#](imtreportapi_cache#type)

There are two types of caches:

-   Permanent: data can only be added but cannot be deleted. It is permanently stored on the server and it is impossible to set time-to-live (TTL) for the data.
-   Temporary: temporary data can be added to this cache, with the explicit specification of the TTL for key-value pairs. After that time data is deleted from the cache.

## Cached data types [#](imtreportapi_cache#data)

The cache allows storing data of three types: key-value pairs, dictionaries and parameters.

Key-value pairs are the main data in the cache. Methods [IMTReportCache::ReadValue\*](/en/docs/mt5/api/reportapi_cache/imtreportcache/imtreportcache_readvalue) and [IMTReportCache::WriteValue\*](/en/docs/mt5/api/reportapi_cache/imtreportcache/imtreportcache_writevalue) are used for working with them.

Dictionaries enable efficient operation with various listings, such as countries, cities, etc. Data in dictionaries is stored as an array of strings. Use methods [IMTReportCache::ReadDictionary\*](/en/docs/mt5/api/reportapi_cache/imtreportcache/imtreportcache_readdictionarypos) and [IMTReportCache::WriteDictionaryString](/en/docs/mt5/api/reportapi_cache/imtreportcache/imtreportcache_writedictionarystring) to work with the dictionary.

In addition, the cache has five optional parameters which the programmer can use for different purposes, for example, to describe the cache contents.

-   From — the beginning of data in the cache. To work with this parameter, use methods [IMTReportCache::ReadParamFrom](/en/docs/mt5/api/reportapi_cache/imtreportcache/imtreportcache_readparamfrom) and [IMTReportCache::WriteParamFrom](/en/docs/mt5/api/reportapi_cache/imtreportcache/imtreportcache_writeparamfrom).
-   To — describes the end of data in the cache. To work with this parameter, use methods [IMTReportCache::ReadParamTo](/en/docs/mt5/api/reportapi_cache/imtreportcache/imtreportcache_readparamto) and [IMTReportCache::WriteParamTo](/en/docs/mt5/api/reportapi_cache/imtreportcache/imtreportcache_writeparamto).
-   String — parameter for storing string data. To work with the parameter use methods [IMTReportCache::ReadParamString](/en/docs/mt5/api/reportapi_cache/imtreportcache/imtreportcache_readparamstring) and [IMTReportCache::WriteParamString](/en/docs/mt5/api/reportapi_cache/imtreportcache/imtreportcache_writeparamstring).
-   Data — parameter for storing binary data. To work with the parameter use methods [IMTReportCache::ReadParamData](/en/docs/mt5/api/reportapi_cache/imtreportcache/imtreportcache_readparamdata) and [IMTReportCache::WriteParamData](/en/docs/mt5/api/reportapi_cache/imtreportcache/imtreportcache_writeparamdata).
-   KeySet — parameter for storing a [set of keys](/en/docs/mt5/api/reportapi_cache/imtreportcachekeyset). To work with the parameter, use methods [IMTReportCache::ReadParamKeySet](/en/docs/mt5/api/reportapi_cache/imtreportcache/imtreportcache_readparamkeyset) and [IMTReportCache::WriteParamKeySet](/en/docs/mt5/api/reportapi_cache/imtreportcache/imtreportcache_writeparamkeyset).

## The general scheme for operations with the cache

-   Create a cache object using the [ReportCacheCreate](/en/docs/mt5/api/imtreportapi/imtreportapi_cache/imtreportapi_reportcachecreate) method
-   Receive cache to it using the [ReportCacheGet](/en/docs/mt5/api/imtreportapi/imtreportapi_cache/imtreportapi_reportcacheget) or [ReportCacheGetTemporary](/en/docs/mt5/api/imtreportapi/imtreportapi_cache/imtreportapi_reportcachegettemporary) method
-   Call [IMTReportCache::ReadBegin](/en/docs/mt5/api/reportapi_cache/imtreportcache/imtreportcache_readbegin) to start reading data. Then use [IMTReportCache::Read\*](/en/docs/mt5/api/reportapi_cache/imtreportcache/imtreportcache_readparamfrom) methods. After finishing data reading call [IMTReportCache::ReadEnd](/en/docs/mt5/api/reportapi_cache/imtreportcache/imtreportcache_readend).
-   Call [IMTReportCache::WriteBegin](/en/docs/mt5/api/reportapi_cache/imtreportcache/imtreportcache_writebegin) to start writing data. Then use [IMTReportCache::Write\*](/en/docs/mt5/api/reportapi_cache/imtreportcache/imtreportcache_writeparamfrom) methods. After finishing data writing call [IMTReportCache::WriteEnd](/en/docs/mt5/api/reportapi_cache/imtreportcache/imtreportcache_writeend). Note that only one report instance can be writing data to cache at a time.

Examples of operations with the data cache are provided in the source code of the Capital report, which is available in "\[Report API installation directory\]\\Report\\Examples\\Capital.Standard.Reports".

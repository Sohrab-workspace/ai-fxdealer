# IMTConClusterState

The IMTConClusterState interface contains methods for controlling network connection between the trading platform components.

Use these methods to receive data on the connection state, network delays and data exchange speed between the current server ([IMTConServer::Id](/en/docs/mt5/api/config_network/imtconserver/imtconserver_id)) and another selected server within the cluster. The selected cluster server is specified during the call of [IMTConServer::ClusterStateGet](/en/docs/mt5/api/config_network/imtconserver/imtconserver_clusterstateget) or [IMTConServer::ClusterStateNext](/en/docs/mt5/api/config_network/imtconserver/imtconserver_clusterstatenext) as an identifier or position in the list of components, respectively.

# Server Reports

The trading platform features multiple reports concerning clients' trading activity, agents' operation, account margin status, gateways and much more. The trading server settings affect what reports are available in the Manager terminal.

When a manager requests a report, the corresponding command is sent to the trading server. The trading server generates a report and sends it to the Manager terminal where it is displayed as a table, HTML page or dashboard.

![Reports](/en/docs/mt5/manager/img/reports.png "Reports")

Using the context menu commands, you can save the report in CSV, Open XML, MHT or HTML file. Diagram settings are available in the context menu for dashboard reports.

-   Title — show/hide the chart title.
-   Legend — show/hide the chart legend.
-   Details — show/hide the chart details.
-   Color — switch color. The option is used for charts displaying information about an entity.
-   Type — switch the chart view: Bar chart, Line chart, Area chart, Donut chart, value (shows the total variable value).
-   Stacking — switch chart stacking type. Used for charts, which compare several entities and their contribution to the overall value. For example, you may distribute positions by symbols and market sentiment. Available options:

-   None — data series are displayed separately
-   With negative values — data series are combined, values are not summed up
-   Regular — data series are combined, values are summed up
-   100% — rows are combined, the general contribution of each series to the total value in percentage is shown

## Requesting a report [#](reports#request)

To request data, click button ![Refresh](/en/docs/mt5/manager/img/refresh_icon.png "Refresh") in the upper right corner of a report.

Reports may have settings: filters by groups, dates and symbols. They can be changed in the upper right corner of the section.

![Report settings](/en/docs/mt5/manager/img/report_settings.png "Report settings")

The availability of the settings depends on the specific report.

-   Groups — group of users, on which the report is generated. Some reports do not support sorting by groups and accounts. When making a request, you may specify:

-   One of the trading groups or subgroups, for example, "demo\\\*.
-   User logins are comma-separated, for example "10011, 10012, 10015".
-   Create or use a previously saved [custom set of accounts](/en/docs/mt5/manager/reports#list).
-   Symbols — symbol, for which the report will be generated (for example, report on operation on a certain symbol). Possibility to filter by symbols depends on the report type.
-   Period — custom period for report generation (for example, report on trading operations for the last week). Dates can be specified both manually and using a calendar that is opened when you press ![Calendar](/en/docs/mt5/manager/img/calendar_icon2.png "Calendar"). Using ![Calendar](/en/docs/mt5/manager/img/calendar_icon.png "Calendar") you can select one of predefined time periods to generate a report for.
-   ![Dashboard](/en/docs/mt5/manager/img/report_dashboard_icon.png "Dashboard") / ![Regular Report](/en/docs/mt5/manager/img/report_old_icon.png "Regular Report") — switch between report types: regular and dashboard. Depending on the implementation of the report by its developer, it can support both types or only one of them.

## Request for reports by client list [#](reports#list)

In order to request for reports conveniently, you can create and use previously saved account sets. Click ![Switch to custom account group request](/en/docs/mt5/manager/img/groups_button.png "Switch to custom account group request") to the right of the Groups field and then click button ![Configuring a group of logins for requesting a report](/en/docs/mt5/manager/img/report_settings_button.png "Configuring a group of logins for requesting a report").

![Creating a group of accounts for requesting reports](/en/docs/mt5/manager/img/report_select_users.png "Creating a group of accounts for requesting reports")

Select the necessary account (or multiple accounts using Shift or Ctrl+Shift) and click Add. In order to add all accounts from a client group, select one account with a necessary group and click Group. Accounts can be added by country the same way.

Specify a name for an account group and click Save. The group will become available in the filter.

A group of accounts can be created based on any csv file (for example, obtained via export in the [Accounts](/en/docs/mt5/manager/accounts) or [Online](/en/docs/mt5/manager/online_accounts) sections). To do this, click Load and select a file. When importing, a login is taken from the file's first data column beginning from the first line, therefore it does not matter how many columns in total are in the file. The program takes the value up to the nearest separator: a space, a tab, a comma or a semicolon. Sample file:

## Report Filters [#](reports#filter)

Using filters, you can display the desired information that meets a specific criteria in your report. For example, you can select operations for a certain instrument or clients with a certain balance or margin, etc.

Filters operate throughout all tabular reports. You can configure them once and apply to any report. To apply a previously created filter, select it from the Filter menu in the list of accounts or clients. To return to the initial list of accounts, click "Not selected".

![Menu for working with filters](/en/docs/mt5/manager/img/report_filter.png "Menu for working with filters")

To create or edit filters, click Customize. The list of all previously created filters is shown in the Filters tab. Click twice on a filter to change its parameters.

![Configuring filters](/en/docs/mt5/manager/img/report_filter_customize.png "Configuring filters")

Specify a filter name and then configure parameters for filtering reports:

-   Time — trading operation or account creation time.
-   Name — name of the account holder.
-   Group — group to which the account belongs.
-   Country — client's country.
-   Account comment — [comment added to the created account](/en/docs/mt5/manager/account_personal). The parameter does not work with comments on trade operations.
-   Previous balance — the previous balance field in [daily reports](/en/docs/mt5/manager/daily_report).
-   Deposit — the amount of deposits into the account.
-   Closed P/L — profit/loss realized from deals.
-   Balance — account balance.
-   Credit — amount of credit on the account.
-   Floating P/L — floating profit/loss of open positions.
-   Equity — account [equity](/en/docs/mt5/manager/account_overview#account_state).
-   Margin — the amount of funds [reserved as margin on the account](/en/docs/mt5/manager/account_overview#margin) for open positions and orders.
-   Free Margin — the amount of [available funds](/en/docs/mt5/manager/account_overview#account_state) on the account.
-   Currency — trading operation currency or account deposit currency.

The filter can be immediately enabled from the editing window by clicking "Apply".

For all text fields, such as Comment, Group, etc., you can use the mask "\*" and the negation sign "!". For example, specifying the comment \*test\* will filter the data for accounts that have the word "test" anywhere in the comment: at the beginning, middle, or end.

### Quick Filters [#](reports#quick_filter)

To quickly filter a tabular report, select any value within it and choose "Filter..." from the context menu. The report will instantly display only the rows containing this value, and the active filter will be shown at the top of the report.

![Quick Filters](/en/docs/mt5/manager/img/report_quick_filter.png "Quick Filters")

## Favorite reports [#](reports#favorites)

The platform provides more than 40 built-in reports. Furthermore, brokers can purchase additional reporting solutions from the [App Store](https://support.metaquotes.net/en/market/mt5/reporting) or develop their own reports using [Report API](https://support.metaquotes.net/en/docs/mt5/api/reportapi). For convenience, the administrator can arrange the reports into directories. However, navigation through so many solutions can be difficult.

To ensure that the reports you need are always at hand, add them to Favorites. Click on the star next to the report name, and it will be added to the corresponding directory. It will still be available in the source directory.

![Selected reports are always at hand](/en/docs/mt5/manager/img/report_favorites.png "Selected reports are always at hand")

## Report cache [#](reports#cache)

Data cache stores data requested from the trading platform database, between report generation times. Use of data cache saves resources when creating reports: the server does not need to retrieve data from the databases again and perform the related calculations, since the data is already available in the cache. However, the data stored in the cache may become outdated. For example, if historical data on the server has significantly changed, it is recommended to reset the report cache to ensure data accuracy and completeness.

To reset the cache, select any report and execute the related command from the report menu. The cache will be cleared for all reports and not only for the selected one.

![Clearing report cache](/en/docs/mt5/manager/img/report_clear_cache.png "Clearing report cache")

-   [Segregated](/en/docs/mt5/manager/segregated_report) — summary report on the change of financial state of the requested accounts for the specified period.
-   [Summary](/en/docs/mt5/manager/summary_report) — summary motion of funds and funds cycles for the selected period.
-   [Gateways Turnover](/en/docs/mt5/manager/gateways_turnover_report) — report on the volume of deals processed by gateways.
-   [Gateways Profit](/en/docs/mt5/manager/gateways_profit_report) — report on the volume of deals processed by gateways, as well as the profit obtained by a broker while processing these trades.

-   [Execution Type](/en/docs/mt5/manager/execution_type_report) — report on the number, volume and type of executed trades.
-   [Trade Accounts](/en/docs/mt5/manager/trade_accounts_report) — report on the current trading statuses of client accounts.
-   [Trade Transactions](/en/docs/mt5/manager/trade_transaction_report) — detailed data on trading operations at MetaTrader 5 servers.
-   [Trade Modifications](/en/docs/mt5/manager/trade_modification_report) — report on trade operations manually changed by an administrator, manager or API.

-   [Trade Performance Summary](/en/docs/mt5/manager/performance_summary_report) — detailed trading statistics of trading accounts.
-   [Trade Group Statistics](/en/docs/mt5/manager/group_statistics_report) —  detailed statistics of groups: the number of accounts, commission amounts, share of trades by type, and more.

-   [StopOut Compensations](/en/docs/mt5/manager/stopouts_compensation_report) — report on operations associated with the compensation of a negative balance in the situation of Stop out. All trades with the "so compensation" type are included into this report.
-   [Money Flow Daily](/en/docs/mt5/manager/money_flow_daily_report) — a group of reports presenting the movement of funds on client accounts.
-   [Money Flow Weekly](/en/docs/mt5/manager/money_flow_weekly_report) — a group of reports representing the weekly movement of funds on client accounts.
-   [Lifetime Value](/en/docs/mt5/manager/lifetime_value_report) — a group of reports concerning client LTVs.
-   [First Time Deposit](/en/docs/mt5/manager/first_time_deposit_report) — a group of reports on first-time deposits on client accounts.
-   [Retention of Clients](/en/docs/mt5/manager/retention_of_clients_report) — new client retention report.
-   [Retention of Trading Accounts](/en/docs/mt5/manager/retention_of_trading_accounts_report) — new client retention report by individual accounts.
-   [NFA](/en/docs/mt5/manager/nfa_report) — a group of reports to be submitted to NFA regulator.
-   [EMIR](/en/docs/mt5/manager/emir_report) — a report to be submitted to regulators according to EMIR.
-   [Fund Overview](/en/docs/mt5/manager/fund_overview_report) — a report reflecting the key characteristics of a hedge fund.
-   [Risk Appetite](/en/docs/mt5/manager/risk_appetite_report) — a report reflecting the risk level accepted by traders.

-   [Fast Profit Deals](/en/docs/mt5/manager/fast_profit_deals_report) — report on profitable deals with very short durations. It enables the detection of dishonest traders exploiting certain situations.

-   [Margin Call Statistics](/en/docs/mt5/manager/margin_call_statistics_report) — accumulated statistics of account margin calls by country.
-   [Accounts Stop Out Statistics](/en/docs/mt5/manager/accounts_stopout_statistics_report) — accounts in the margin call or stop-out state along with detailed information.
-   [Clients Statistics](/en/docs/mt5/manager/clients_statistics_report) — comprehensive client statistics, including type, registration status, source of income, etc.
-   [Accounts Assets and Liabilities](/en/docs/mt5/manager/accounts_assets_liabilities_report) — distribution of client account assets and liabilities by country.
-   [Traders Subscriptions](/en/docs/mt5/manager/traders_subscriptions_report) — statistics on the popularity of subscriptions to broker services.
-   [Autotrading Statistics](/en/docs/mt5/manager/autotrading_statistics_report) — provides insight into user engagement with trading robots and signals.
-   [Trading Frequency](/en/docs/mt5/manager/trading_frequency_report) — allows you to analyze the number of deals per user over time.
-   [Clients Activity](/en/docs/mt5/manager/clients_activity_report) — comparative statistics on active and inactive clients.
-   [A-Book & B-Book Exposure](/en/docs/mt5/manager/abook_bbook_exposure_report) — total volume of clients' current positions routed to external liquidity providers (A-Book) and those executed internally by the broker (B-Book).
-   [Execution Speed](/en/docs/mt5/manager/execution_speed_report) — frequency of order placement on client accounts.

The availability of these reports is determined by an administrator when setting up the platform.

## Additional reports [#](reports#additional)

A wide variety of additional reports is available in the [App Store](https://support.metaquotes.net/en/market/mt5/reporting) on the technical support website. The App Store showcase is featured directly in Manager Terminals, allowing managers to view and order additional products without leaving their workspace:
